# Security Policy

## Supported Versions

Only the latest release is supported for security fixes. Older releases should be upgraded to the latest available version before requesting support.

| Version | Supported |
| --- | --- |
| Latest release | Yes |
| Older releases | No |

## Reporting a Vulnerability

Please use GitHub Private Vulnerability Reporting through the repository Security tab and the `Report a vulnerability` flow.

- Do not open public GitHub issues for security vulnerabilities.
- Include clear reproduction steps, affected version information, and impact details where possible.
- Reports are reviewed on a best-effort basis by the project maintainer as time permits.

## What to Expect

- Best-effort acknowledgement and investigation
- A request for more information if needed
- Coordination on disclosure timing when a report is confirmed

There is no guaranteed SLA for response or remediation timing.
