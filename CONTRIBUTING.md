# Contributing

Thanks for your interest in AeroDocs.

AeroDocs is a maintainer-led project. Issues for bugs and feature requests are welcome. Pull requests are generally discouraged unless they are small, obvious fixes or the project maintainer has asked for one first.

## Before You Start

- Please open an issue before doing non-trivial work.
- Unsolicited pull requests may be closed at maintainer discretion.
- Small fixes such as typo corrections, broken links, and straightforward documentation cleanups are usually fine without prior discussion.

## Issues

- Use GitHub issues for bug reports and feature requests.
- Please keep one topic per issue where practical.
- For security vulnerabilities, do not open a public issue. Follow [SECURITY.md](SECURITY.md) instead.

## Pull Requests

- If you want to submit a non-trivial change, open an issue first and wait for maintainer feedback before writing code.
- PRs that arrive without prior discussion may be closed without review.
- Even discussed PRs are not guaranteed to be accepted.
- Please keep accepted PRs narrow in scope and easy to review.

## Licensing

By submitting a contribution that is accepted into this repository, you agree that your contribution will be licensed under the repository's Apache License 2.0 terms. No separate CLA is required at this time.

## Support Expectations

This is a personal project maintained on a best-effort basis. Issue triage, PR review, and support responses happen as time permits.
